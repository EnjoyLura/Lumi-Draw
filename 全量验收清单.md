# 露米绘画全量验收清单

> 本清单用于全部功能开发完成后的最终验收。界面验收以 `prototype/mobile-prototype.html` 和 `prototype/admin-prototype.html` 为唯一 UI 依据；接口与业务验收以真实后端数据为准，同时保留 mock 开关回归。

## 验收前置

- 本地依赖已安装，根目录可执行 `corepack pnpm ...`。
- 后端 `.env` 已配置 PostgreSQL、Redis、OSS、KIE、微信登录、微信支付相关变量。
- 测试环境允许 mock 登录、mock 生成和 mock 支付；生产验收前必须关闭 `AUTH_ALLOW_MOCK_LOGIN`、`GENERATE_ALLOW_MOCK`、`PAYMENT_ALLOW_MOCK`。
- 生产上线前必须执行 `ENV_FILE=apps/api/.env pnpm check:production-safety`，确认默认管理员密码、mock 开关、CORS、KIE、OSS、微信登录和微信支付配置全部达标。
- 小程序和管理后台分别验证“模拟数据开”和“模拟数据关”两种状态。
- 每轮验收前确认 `git status --short` 干净，且不包含 `agent.md`、`.env*` 等敏感文件。

## 自动化检查

本地静态检查：

```bash
corepack pnpm verify:static
```

覆盖范围：
- monorepo TypeScript 类型检查。
- NestJS 后端构建。
- React 管理后台构建。
- uni-app H5 构建。
- Git 空白字符检查。

后端接口 smoke：

```bash
API_BASE=http://127.0.0.1:3000/api ADMIN_USERNAME=admin ADMIN_PASSWORD=admin123 SMOKE_GENERATE_MOCK=true pnpm verify:api
```

部署到服务器后，在服务器项目目录执行同一条接口 smoke。公网 HTTPS 在本机不可访问时，以服务器本机 `http://127.0.0.1:3000/api` 结果为准。

管理后台真实 UI smoke：

```bash
ADMIN_REAL_API_TARGET=http://122.51.235.145:3000/api ADMIN_USERNAME=admin ADMIN_PASSWORD=admin123 pnpm smoke:admin-real-ui
```

覆盖范围：
- 管理后台关闭模拟数据后使用真实管理员 token 登录。
- 工作台、用户管理、用户详情、作品管理、作品详情、运营配置、走马灯管理真实渲染。
- 用户积分调整、作品精选/推荐切换、走马灯启停通过 UI 发起并由真实 API 反查持久化。
- 退出登录清理管理员 token 并回到登录页。

## 接口测试

公共配置：
- `GET /health` 返回服务健康状态。
- `GET /app/bootstrap` 返回首页、创作、支付、会员、运营配置所需基础数据。
- `GET /config/*` 各配置接口返回可用列表或协议内容。

用户与鉴权：
- 微信静默登录成功后可获取 accessToken 与 refreshToken。
- accessToken 过期后 refresh 可轮换。
- 退出登录后需要登录的接口返回统一鉴权错误。
- `GET/PATCH /users/me` 可读取和更新昵称、头像、简介、性别、手机号。

作品与社交：
- 作品列表、广场、搜索、详情、我的作品、草稿箱分页正常。
- 发布、编辑、删除、公开状态变更能持久化。
- 点赞、收藏、关注、浏览记录、反推同款计数真实持久化。
- 用户主页、关注列表、收藏列表、浏览记录与互动状态一致。
- 举报提交后后台可见并可处理。

积分、支付与会员：
- 积分余额与流水一致。
- 签到只允许每日一次，连续签到奖励正确。
- 邀请绑定防止重复绑定和自我绑定，奖励入账。
- 充值订单、会员订单状态可查询；mock 支付完成后到账。
- 生产支付配置齐全时，微信支付参数、回调验签、幂等到账流程可用。

AI 生成：
- 文生图、图生图、反推提示词可创建任务。
- 任务查询、取消、重试、失败退积分、成功自动保存草稿闭环可用。
- KIE 回调优先更新任务状态；轮询查询作为兜底。
- 生成结果转存 OSS 后展示使用转存地址。

消息与反馈：
- 系统消息、互动消息、生成消息、公告可读取和标记已读。
- 用户反馈可提交，后台可处理和回复。
- 后台推送发送后用户侧可见。

管理后台：
- 管理员登录、退出、鉴权拦截正常。
- 仪表盘、用户管理、作品管理、审核、举报、反馈、交易记录、运营配置、财务配置、系统配置均可真实读写。
- 模拟数据关闭后，页面不依赖静态 mock 数据完成核心操作。

## 功能测试

小程序主流程：
- 首次进入首页，推荐/最新切换、下拉刷新、上滑加载、banner、玩法入口、作品入口正常。
- 从首页进入创作页，选择玩法、模型、比例、清晰度、数量，提交生成任务。
- 任务进行中可在创作页和画廊看到状态；任务完成后提示作品已自动保存为草稿，并能回到创作页查看结果。
- 草稿可编辑、发布；发布后在作品详情、画廊、用户主页按状态展示。
- 广场分类、收藏 Tab、作品卡片作者入口、详情入口正常。
- 搜索热门词、历史词、结果分页、空状态正常。
- 作品详情内点赞、收藏、关注、复制提示词、长按图片、举报、重新生成、一键同款正常。
- 我的页登录态、未登录态、资料编辑、设置、消息、反馈、积分、签到、邀请、会员、充值入口正常。

管理后台主流程：
- 模拟数据开关能在不影响登录状态的情况下切换数据源。
- 仪表盘、数据详情可查看真实统计。
- 用户详情可封禁、解封、调整积分。
- 作品详情可精选、推荐、下架、恢复。
- 内容审核、举报、反馈处理后状态同步。
- 运营配置新增、编辑、删除后小程序配置接口同步变化。
- 财务配置变更后小程序充值、会员、签到、邀请展示同步变化。
- 公告、推送、敏感词、版本、协议、审核设置可保存并生效。

异常与边界：
- 未登录访问需要登录的动作时弹出登录引导。
- 接口失败时页面有明确反馈，且不破坏当前页面状态。
- 分页无更多数据、空列表、弱网重试、重复点击提交都有保护。
- 生成失败、取消、部分成功、支付失败、审核拒绝都有可恢复路径。

## 界面测试

小程序：
- 逐页对照 `prototype/mobile-prototype.html`，确认页面元素、交互状态、跳转关系、弹窗、抽屉、加载、切换动画没有漏做或多做。
- 在 H5 验收核心路径，并在微信开发者工具中验收小程序端差异。
- 验证一级 Tab、二级页进入/返回、抽屉、弹窗、复制、长按、上滑加载的交互连贯。
- 验证未登录、已登录、空状态、加载中、错误状态、生成中、生成成功、生成失败各状态。

管理后台：
- 逐模块对照 `prototype/admin-prototype.html`，确认路由、列表、详情、表单、弹窗、操作反馈一致。
- 验证模拟数据开关、登录门、退出登录和真实接口错误提示。
- 验证桌面浏览器常用宽度下列表、表单、弹窗、侧栏、顶部栏无内容遮挡。

## 通过标准

- `corepack pnpm verify:static` 通过。
- 服务器 `pnpm verify:api` 通过。
- `pnpm smoke:admin-real-ui` 通过。
- 生产上线前 `ENV_FILE=apps/api/.env pnpm check:production-safety` 通过。
- 小程序所有页面在模拟数据开/关两种状态下主流程可完成。
- 管理后台模拟数据关闭后核心管理能力可完成真实读写。
- 原型对照没有影响交付的漏项、错跳转、错状态或明显不一致交互。
- 生产上线前敏感配置不在 Git 中，mock 登录、mock 生成、mock 支付按环境要求关闭。

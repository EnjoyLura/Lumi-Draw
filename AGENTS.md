# Lumi-Draw Agent 开发规则

## 项目定位

本项目是《露米绘画AI》微信 AI 画图小程序及配套管理后台。主体为个体工商户，前端实现必须以 `prototype/mobile-prototype.html` 和 `prototype/admin-prototype.html` 为唯一产品原型来源。

## 核心原则

- 原型图就是答案。开发任务是把原型迁移到目标技术栈，不重新设计产品。
- 所有文档、任务说明和代码注释不得写前端样式方案；实现界面时直接对照原型文件逐项复刻。
- 遵守最小原则。每次任务只完成一个一级界面；二级界面必须单独成任务。
- 开发顺序固定为前端优先：先在本地 Codex 开发小程序前端和管理后台前端，调整满意后再通过 VS Code Remote SSH + Codex 插件在服务器开发后端并联调。
- 保持代码干净、解耦、可维护，避免为短期交付制造臃肿结构。
- 遇到必须由用户确认的信息，先停止并提问，不做高风险猜测。

## 项目结构约定

后续采用 monorepo：

- `apps/mobile`：uni-app 微信小程序，Vue 3 + TypeScript。
- `apps/api`：NestJS 后端，Node.js + TypeScript。
- `apps/admin`：React + Ant Design Pro 管理后台。
- `prototype`：产品原型，作为界面和交互还原依据。

当前第一阶段只维护文档和安全基线，不搭建脚手架。后续本地 Codex 阶段只开发 `apps/mobile` 和 `apps/admin` 的前端；`apps/api` 在服务器 Remote SSH 阶段创建和开发。

## 敏感配置规则

- 真实密钥、AppSecret、支付密钥、OSS 密钥、KIE API Key、服务器账号等只允许写入本地 `agent.md` 或 `.env*`。
- `agent.md` 和 `.env*` 必须保持 Git 忽略状态，禁止提交。
- 可提交文档只允许描述变量名和用途，不允许出现真实值。
- 提交前必须运行 `git status --short`，确认没有敏感配置进入待提交列表。

## 开发流程

1. 开始每个界面任务前，先通读对应原型代码和相关交互函数。
2. 明确本次任务只包含一个界面，不顺手迁移其他界面。
3. 本地前端阶段必须使用 mock 数据开关保证小程序和管理后台都能完整体验流程。
4. 小程序前端和管理后台前端全部调整满意前，不在本地 Codex 阶段开发后端实现。
5. 完成后自主验证界面内容、交互、跳转、状态和原型一致，没有漏做或多做。
6. 验证通过后进行一次 Git 提交。
7. 每完成一个界面后，部署到公网环境供用户浏览验收；用户确认满意后再继续下一界面。
8. 前端验收完成后，再由用户使用 VS Code Remote SSH 打开服务器工作区，并使用 Codex 插件开发 NestJS 后端和前后端联调。

## 验证要求

- 小程序界面任务：至少完成构建检查、核心交互手测、原型对照清单。
- 管理后台前端任务：至少完成构建检查、路由/表单/列表/操作反馈验证，并确认设置中的模拟数据开关可用。
- 后端任务：至少完成单元或接口级验证，并确认 mock 开关关闭时能走真实 API。
- 部署任务：确认公网域名可访问，必要时给出访问地址和当前版本说明。

## 原型页面索引

小程序原型主要页面：

- 一级入口：露米绘画、广场、创作、画廊、我的。
- 二级页面：全部玩法、搜索、反推提示词、积分充值、每日签到、邀请好友、会员中心、消息、消息详情、设置、编辑资料、意见反馈、发布作品、编辑作品、草稿箱、更新日志、浏览记录、用户主页、关注列表、切换模型、举报、作品详情。

管理后台原型主要模块：

- 仪表盘：工作台、数据大屏、数据详情。
- 用户与内容：用户管理、用户详情、作品管理、作品详情、内容审核、审核详情。
- 运营配置：走马灯、玩法模板、风格管理、分类管理、热搜管理、模型管理、分辨率配置、尺寸比例。
- 财务管理：财务概览、充值方案、会员方案、签到配置、邀请配置、交易记录、积分基础配置。
- 消息管理：弹窗公告、系统通知、用户反馈。
- 系统：系统设置、审核设置、敏感词管理、版本管理、协议管理。

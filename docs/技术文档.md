# Lumi-Draw 技术文档

## 1. 系统架构

### 1.1 整体架构
```
用户 -> 微信小程序(uni-app H5/WeChat) -> Nginx(80)
管理员 -> React管理后台(Web)            -> Nginx(80)
                                              |
                                    +---------+---------+
                                    |                   |
                                /api/*             /admin/* 和 /
                                    |                   |
                              NestJS Server(3000)    静态文件
                                    |
                    +-------+-------+-------+
                    |       |       |       |
                PostgreSQL Redis  OSS   KIE.AI
                 (5432)   (6379) (外部)  (外部)
```

### 1.2 部署架构
```yaml
docker-compose:
  postgres:     # PostgreSQL 16
  redis:        # Redis 7
  server:       # NestJS 后端 (Node 22)
  nginx:        # Nginx 反向代理
```

- 瀑布流分页查询

- 浏览记录

#### payment 支付模块

#### admin 管理后台模块

- 运营配置 CRUD（Banner/玩法/风格/热搜词）

  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  openid VARCHAR(64) UNIQUE NOT NULL,
  union_id VARCHAR(64),
  nickname VARCHAR(20) DEFAULT '',
  avatar_url TEXT DEFAULT '',
  gender SMALLINT DEFAULT 0,           -- 0未知 1男 2女

## 4. Redis 用途

| Key 模式 | 用途 | TTL |
|----------|------|-----|
| `wx:access_token` | 微信 AccessToken 缓存 | 7200s |
| `user:session:{id}` | 用户会话信息 | 7d |
| `checkin:today:{userId}` | 今日是否已签到 | 当日剩余 |
| `checkin:streak:{userId}` | 连续签到天数 | 30d |
| `ranking:works:likes` | 作品点赞排行榜 | 实时更新 |
| `generate:task:{taskId}` | 生成任务状态缓存 | 1h |

## 5. BullMQ 任务队列

### 5.1 队列定义
```typescript
// 队列名称
const GENERATE_QUEUE = 'generate';

// 任务类型
const GENERATE_TASK = 'generate-image';
const REVERSE_PROMPT_TASK = 'reverse-prompt';
```

### 5.2 生成任务流程
```
前端提交 -> Controller校验 -> 扣减积分 -> 添加BullMQ任务
-> Worker处理:
  1. 调用 KIE.AI POST /api/v1/jobs/createTask
  2. 保存 kieTaskId
  3. 等待 callback 或主动轮询
  4. 下载结果图片 -> 上传 OSS
  5. 更新 generation 状态
  6. 创建 work 记录(草稿)
```

### 5.3 Callback 处理
```
KIE.AI POST /api/generate/callback
-> 解析 taskId + resultJson
-> 更新 generation 记录
-> 下载图片 -> 上传 OSS
-> 更新 result_urls
```

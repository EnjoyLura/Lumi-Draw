# Lumi-Draw 产品需求文档 (PRD)

## 1. 产品概述

### 1.1 产品定位
Lumi-Draw（露米绘画AI）是一款面向C端用户的微信小程序AI绘画工具，通过多模型AI图像生成能力，让用户以低门槛、高趣味性的方式创作AI艺术作品。

### 1.2 目标用户
- 18-35岁对AI绘画感兴趣的年轻用户
- 社交媒体内容创作者
- 头像/壁纸等个性化图片需求用户

### 1.3 核心价值
- 多模型选择（GPT Image 2 / Nano Banana 2 / Nano Banana Pro / Seedream 4.5）
- 积分体系降低使用门槛（签到/邀请/充值多渠道获取）
- 玩法模板一键创作，降低学习成本
- 社区广场分享交流

## 3. 核心业务流程

### 3.1 用户登录流程
```
用户打开小程序 -> wx.login获取code -> 后端用code换openid -> 查找/创建用户 -> 返回JWT Token -> 前端存储Token
```

### 3.2 AI 绘图流程
```
创作页选择参数 -> 点击"开始创作" -> 校验积分余额 -> 扣减积分 -> 创建生成记录(pending)
-> 提交BullMQ任务 -> Worker调用KIE.AI createTask -> 等待callback
-> callback接收结果 -> 更新生成记录(success) -> 图片上传OSS -> 前端轮询/WebSocket获取结果
```

### 3.3 作品发布流程
```
画廊选择草稿 -> 填写标题/描述/标签 -> 提交发布 -> 微信内容安全审核
-> 开启人工审核: 进入审核队列(待审核) -> 管理员审核通过/拒绝
-> 未开启人工审核: 直接发布到广场
```

## 4. 内容审核策略

### 4.1 MVP 阶段
- 使用微信小程序自带内容安全接口（`security.msgSecCheck` / `security.mediaCheckAsync`）
- 管理后台可开关「人工审核」
  - 开启：用户发布作品需管理员审核后公开
  - 关闭：通过微信内容安全后直接公开

### 4.2 举报机制
- 举报原因：垃圾广告/色情低俗/违法违规/侵权盗版/煽动仇恨/虚假信息/其他
- 管理后台举报管理模块处理

## 5. 数据模型概览

### 5.1 核心实体
- **用户(User)**: 微信openid、昵称、头像、性别、签名、积分余额、会员状态
- **作品(Work)**: 标题、描述、图片URL、模型、风格、比例、状态（草稿/待审核/已发布/已下架）
- **生成记录(Generation)**: 关联用户、prompt、模型、参数、KIE任务ID、状态、结果图
- **交易记录(Transaction)**: 用户、类型（收入/支出）、渠道（充值/创作/签到/会员）、积分变动、金额
- **模型(AIModel)**: 名称、描述、积分消耗、标签、封面、启用状态
- **风格(Style)**: 名称、示例图、排序
- **Banner**: 标题、图片URL、跳转链接、启用状态、排序
- **玩法模板(Gameplay)**: 名称、封面、使用人数、热门标记
- **热搜词(HotSearch)**: 关键词、搜索量、启用状态
- **充值方案(RechargeTier)**: 价格、积分、赠送、热门标记
- **会员方案(MemberPlan)**: 名称、价格、权益描述
- **签到里程碑(CheckinMilestone)**: 连续天数、奖励积分
- **公告(Notification)**: 标题、内容、封面、按钮、跳转、推送范围、弹窗开关、发布状态
- **反馈(Feedback)**: 用户、类型、描述、截图、微信号、状态
- **举报(Report)**: 举报者、目标、原因、描述、状态

### 5.2 关联关系
- User 1:N Work（一个用户多个作品）
- User 1:N Generation（一个用户多次生成）
- User 1:N Transaction（一个用户多条交易）
- User N:N User（关注/粉丝）
- Work N:N Tag（作品标签）
- User 1:N Feedback / Report

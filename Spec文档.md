# 露米绘画AI Spec 文档

## 1. 开发总原则

- 最小原则：每次任务只做一个明确交付物。
- 原型优先：实现前必须阅读对应原型代码，原型是界面和交互的唯一依据。
- 不重新设计：不得主动新增原型没有的功能、入口、状态或交互。
- 解耦推进：基础框架、一级界面、二级界面、后端能力、联调和部署分阶段完成。
- 每个任务完成后必须验证、提交 Git，并部署给用户公网验收。

## 2. 任务拆分规则

### 2.1 第一阶段：文档与安全基线

- 建立 `.gitignore`，防止密钥提交。
- 建立 `AGENTS.md`、`产品需求文档.md`、`技术文档.md`、`Spec文档.md`。
- 建立本地 `agent.md` 保存私有配置，但不提交。

### 2.2 第二阶段：基础框架

- 创建 monorepo 基础结构。
- 创建小程序、后端、管理后台三个项目的最小可运行框架。
- 建立共享类型和 API client 约定。
- 建立 mock 数据开关和环境变量读取机制。

### 2.3 第三阶段：小程序界面迁移

每次只迁移一个界面。推荐顺序：

1. 首页。
2. 广场。
3. 创作。
4. 画廊。
5. 我的。
6. 按用户确认顺序迁移二级页面。

每个界面任务必须包含：

- 阅读对应原型页面结构和交互函数。
- 迁移该界面所需数据结构和 mock 数据。
- 接入统一 API client，不直接散落请求。
- 完成原型对照自查。
- 提交 Git。
- 部署公网验收。

### 2.4 第四阶段：后端能力开发

后端按业务闭环拆分：

1. 微信静默登录和用户初始化。
2. 作品与草稿基础能力。
3. AI 生成任务和 KIE 回调。
4. OSS 上传与结果转存。
5. 积分、充值和微信支付。
6. 审核、举报和人工审核开关。
7. 消息、公告和反馈。
8. 管理后台 API。

### 2.5 第五阶段：管理后台迁移

每次只迁移一个后台页面或一个详情页。优先从工作台开始，再按用户与内容、运营配置、财务、消息、系统模块推进。

## 3. Mock 数据开关

- 小程序和管理后台都必须保留 mock 数据开关。
- 开关开启：使用本地 mock 数据，可完整体验前端流程。
- 开关关闭：使用真实后端 API，用于联调和生产验证。
- 页面组件不得直接判断大量 mock 分支，统一由数据服务层切换。
- mock 数据结构必须尽量贴近真实 API 返回结构。

## 4. API 契约原则

- 前端和后台通过同一后端 API 交互。
- 先定义请求和响应类型，再实现页面接入。
- 后端错误返回统一包含错误码、错误消息和可选详情。
- 分页列表统一返回列表、总数、页码和页大小。
- 涉及支付、积分、生成任务、回调的接口必须具备幂等处理。

## 5. AI 生成流程

标准流程：

1. 用户选择模型、玩法、风格、精度、比例和数量。
2. 用户输入提示词或上传参考图。
3. 前端提交生成请求。
4. 后端校验用户、积分、模型和参数。
5. 后端创建本地生成任务并扣减或冻结积分。
6. 后端通过 BullMQ 调用 KIE 创建任务。
7. KIE 回调或后端兜底查询任务状态。
8. 后端转存结果到 OSS。
9. 后端更新任务状态、作品草稿和积分流水。
10. 前端展示生成结果，用户可发布、编辑或继续生成。

## 6. 支付与积分

- 积分是 AI 生成的主要消耗单位。
- 微信支付用于购买积分或会员。
- 积分流水必须记录来源、用途、变动数量、关联订单或任务。
- 支付回调必须幂等，避免重复到账。
- 生成失败时按后端策略退回或释放积分。

## 7. 审核与风控

- MVP 使用微信内容审核能力加人工审核开关。
- 用户发布作品时按配置进入待审核或直接发布。
- 管理后台可处理待审核作品和举报。
- 敏感词、人工审核开关、审核状态必须由后台配置维护。

## 8. 图标资源原则

- 小程序不依赖运行时 CDN 图标库。
- 后续实现时优先使用本地化 iconfont 或 uni-icons。
- 原型中的 Remix Icon 名称应建立映射表，缺失图标再由用户补充 iconfont 素材。

## 9. 验收清单

每个界面任务完成后必须检查：

- 页面入口和返回路径正确。
- 原型中的按钮、输入、列表、弹层、状态反馈和跳转已覆盖。
- 没有迁移本任务以外的界面。
- mock 开关开启时能完整体验当前界面。
- mock 开关关闭时不破坏真实 API 接入约定。
- 构建或类型检查通过。
- Git 只包含本任务相关变更。

## 10. 提交与部署规则

- 每个任务完成后只提交一次相关变更。
- 提交前检查 `git status --short`。
- 禁止提交 `agent.md`、`.env*`、密钥、证书和本地缓存。
- 每个界面任务完成后部署前后端到公网，让用户通过浏览器或小程序开发工具验收。
- 用户验收通过后再开始下一任务。

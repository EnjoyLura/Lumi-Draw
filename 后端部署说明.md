# 后端开发与部署说明

## 当前后端阶段

当前已创建 `apps/api` NestJS 基础服务，提供：

- `/api/health`
- `/api/docs`
- 环境变量校验
- 全局响应包裹
- 全局异常格式
- Dockerfile
- `docker-compose.yml` 中的 API、PostgreSQL、Redis 基线

## 本地命令

```bash
pnpm install
pnpm --filter @lumi-draw/api typecheck
pnpm --filter @lumi-draw/api build
pnpm --filter @lumi-draw/api start
```

健康检查：

```bash
curl http://localhost:3000/api/health
```

Swagger：

```bash
http://localhost:3000/api/docs
```

## 服务器部署目录

服务器项目目录：

```text
/home/ubuntu/Lumi-Draw
```

## 服务器环境变量

服务器必须创建：

```text
apps/api/.env
```

参考：

```bash
cp apps/api/.env.example apps/api/.env
```

真实密钥只写服务器 `.env`，不要提交 Git。

生产必填：

- `DATABASE_URL`
- `REDIS_URL`
- `JWT_SECRET`
- `ADMIN_JWT_SECRET`
- `CALLBACK_SECRET`
- `WX_APPID`
- `WX_APPSECRET`
- `WX_MCH_ID`
- `WX_PAY_API_V3_KEY`
- `WX_PAY_CERT_SERIAL_NO`
- `WX_PAY_PRIVATE_KEY` 或 `WX_PAY_PRIVATE_KEY_PATH`
- `WX_PAY_PLATFORM_CERTIFICATE` 或 `WX_PAY_PLATFORM_CERTIFICATE_PATH`
- `WX_PAY_NOTIFY_URL`
- `KIE_API_KEY`
- `KIE_CALLBACK_URL`
- `OSS_ACCESS_KEY_ID`
- `OSS_ACCESS_KEY_SECRET`
- `OSS_BUCKET`
- `OSS_ENDPOINT`

## 国内服务器镜像建议

如果服务器拉取依赖慢：

```bash
pnpm config set registry https://registry.npmmirror.com
```

如果 Docker 拉取镜像慢，优先配置服务器 Docker 镜像源，再执行 compose。

## Docker Compose 启动

```bash
docker compose up -d --build api
docker compose ps
docker compose logs -f api
```

健康检查：

```bash
curl http://127.0.0.1:3000/api/health
```

## Nginx 反向代理目标

将公网 `/api/` 转发到：

```text
http://127.0.0.1:3000/api/
```

KIE 回调地址：

```text
https://ejoyflie.cloud/api/generate/callback
```

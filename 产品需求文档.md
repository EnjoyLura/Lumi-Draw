# 露米绘画AI 产品需求文档

## 1. 产品目标

《露米绘画AI》是面向微信小程序用户的 AI 画图产品，提供作品发现、AI 创作、个人画廊、积分付费、会员权益、内容发布和运营管理能力。管理后台用于运营人员维护模型、玩法、内容审核、财务和消息配置。

产品实现以 `prototype/mobile-prototype.html` 和 `prototype/admin-prototype.html` 为唯一界面与交互依据。前端任务是原型迁移，不重新设计。

## 2. 用户角色

- 普通用户：浏览作品、使用 AI 模型生成图片、发布作品、管理个人画廊、充值积分、开通会员、签到和邀请。
- 创作者用户：管理草稿和已发布作品，查看互动消息，维护个人资料。
- 运营管理员：管理用户、作品、审核、模型、玩法、积分、会员、交易、公告、通知和反馈。

## 3. 微信小程序范围

### 3.1 一级入口

- 首页：展示公告/轮播、热门玩法、推荐作品和最新作品。
- 广场：展示社区作品流，支持分类、搜索、筛选、互动和作品详情进入。
- 创作：选择玩法、模型、风格、图片精度、画面比例、生成数量，输入提示词并提交生成。
- 画廊：展示当前用户资料、作品、草稿和个人作品管理入口。
- 我的：展示用户账户、积分、会员、常用功能入口和设置入口。

### 3.2 二级页面

- 全部玩法：展示全部玩法模板，并能套用到创作流程。
- 搜索：支持历史搜索、热搜和作品搜索结果。
- 反推提示词：上传图片后生成提示词，并带回创作页。
- 积分充值：展示积分余额、充值档位、收入/消费记录。
- 每日签到：展示签到状态、连续签到和里程碑奖励。
- 邀请好友：展示邀请奖励规则和邀请记录。
- 会员中心：展示会员方案、权益和开通入口。
- 消息与消息详情：展示系统通知、互动、审核等消息分类和明细。
- 设置：提供主题、模拟数据开关、账户和系统相关入口。
- 编辑资料：维护头像、昵称、性别、签名和露米号。
- 意见反馈：提交问题类型、文字说明和图片。
- 发布作品、编辑作品、草稿箱：完成作品发布、资料编辑和草稿管理。
- 更新日志、浏览记录、用户主页、关注列表、切换模型、举报、作品详情：按原型提供对应浏览和操作能力。

## 4. 管理后台范围

### 4.1 仪表盘

- 工作台：展示待办、关键指标、快捷入口和趋势概览。
- 数据大屏：展示用户、作品、收入等核心趋势。
- 数据详情：查看单项指标明细。

### 4.2 用户与内容

- 用户管理：搜索、筛选、查看用户列表。
- 用户详情：查看资料、积分、会员、交易、作品和管理操作。
- 作品管理：查看作品列表、状态、推荐、编辑和下架。
- 作品详情：查看作品信息、生成参数、互动数据和管理操作。
- 内容审核：处理待审核作品和举报。
- 审核详情：查看审核对象并通过、拒绝或处理举报。

### 4.3 运营配置

- 走马灯管理：维护首页轮播内容、跳转和启用状态。
- 玩法模板：维护玩法名称、提示词模板、启用状态和排序。
- 风格管理：维护创作风格选项。
- 分类管理：维护广场分类。
- 热搜管理：维护搜索页热搜词。
- 模型管理：维护 AI 模型名称、能力、积分消耗和上下线。
- 分辨率配置：维护图片精度档位。
- 尺寸比例：维护画面比例选项。

### 4.4 财务管理

- 财务概览：展示收入、充值、会员和消耗情况。
- 充值方案：维护充值金额、到账积分和赠送积分。
- 会员方案：维护会员价格、权益、赠送积分和签到加成。
- 签到配置：维护每日签到和连续签到奖励。
- 邀请配置：维护邀请奖励。
- 交易记录：查看充值、消费、退款等记录。
- 积分基础配置：维护积分消耗和奖励基础规则。

### 4.5 消息与系统

- 弹窗公告：维护首页公告内容和展示状态。
- 系统通知：向指定用户群发送通知。
- 用户反馈：查看、回复和处理反馈。
- 系统设置：维护模拟数据、审核、协议、版本等系统配置入口。
- 审核设置：维护自动审核和人工审核开关。
- 敏感词管理：维护敏感词列表。
- 版本管理：维护更新日志。
- 协议管理：维护用户协议和隐私政策。

## 5. 核心业务规则

- 登录方式仅微信静默登录，前端通过 `wx.login` 换取后端会话。
- 开发阶段必须保留模拟数据开关；开启时走 mock 数据，关闭时走真实后端 API。
- AI 生成采用任务模式：创建任务、等待回调或查询状态、保存结果。
- 用户生成作品默认进入草稿或生成记录，发布后进入广场展示并接受审核策略控制。
- 付费方案为微信支付加积分体系，AI 生成按模型、精度、数量扣减积分。
- 内容审核 MVP 使用微信自带接口加人工审核开关。
- 管理后台为独立 Web 项目，通过同一后端 API 交互。

## 6. 验收原则

- 每次只交付一个界面或一个明确后端能力。
- 界面完成后必须对照原型检查功能、交互、跳转和状态，不新增原型外功能。
- 每次任务完成后提交 Git，并部署到公网供用户验收。
- 用户确认当前交付质量后，再进入下一任务。

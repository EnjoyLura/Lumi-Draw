# 露米绘画AI 技术文档

## 1. 技术栈

- 小程序：uni-app，Vue 3，TypeScript，微信小程序目标平台。
- 后端：NestJS，Node.js，TypeScript。
- 管理后台：React，Ant Design Pro，TypeScript，独立 Web 项目。
- 数据库：PostgreSQL。
- 缓存与队列：Redis，BullMQ。
- 对象存储：阿里云 OSS，bucket 为 `lumidraw`，区域为 `oss-cn-beijing`。
- AI 平台：KIE.AI。
- 支付：微信支付。
- 部署：docker-compose。

## 2. Monorepo 规划

后续目录规划：

- `apps/mobile`：微信小程序。
- `apps/api`：NestJS API 服务。
- `apps/admin`：管理后台。
- `packages/shared`：跨端共享类型、常量和接口契约。
- `prototype`：原型文件，仅作为迁移和验收依据。
- `docs` 或根目录文档：产品、技术和 Spec 文档。

第一轮不创建脚手架，仅建立文档和安全规则。

## 3. 配置与密钥

- 私有配置只放 `agent.md` 或 `.env*`，并由 `.gitignore` 忽略。
- 可提交代码和文档只使用变量名，不写真实密钥。
- 后端启动时从环境变量读取微信、KIE、OSS、支付、数据库、Redis 配置。
- 生产环境由服务器环境变量或部署平台注入，不从 Git 仓库读取真实密钥。

关键变量名：

- 微信：`WX_APPID`、`WX_APPSECRET`。
- KIE：`KIE_API_BASE`、`KIE_API_KEY`、`KIE_CALLBACK_URL`。
- OSS：`OSS_ACCESS_KEY_ID`、`OSS_ACCESS_KEY_SECRET`、`OSS_BUCKET`、`OSS_ENDPOINT`。
- 微信支付：`WX_MCH_ID`、`WX_MCH_API_KEY`。

## 4. 小程序技术边界

- 登录只使用微信静默登录：小程序调用 `wx.login`，后端使用 code 换取 openid/session。
- 小程序侧保留 mock 数据开关，开关开启时所有页面从本地 mock 服务层读取数据。
- 小程序页面只通过统一 API client 或 mock client 获取数据，页面内不散落请求逻辑。
- 原型中的图标后续迁移为本地化 iconfont/uni-icons 方案，不依赖运行时 CDN。
- 图片上传通过后端签名或后端代理进入 OSS，避免小程序端持有 OSS 密钥。

## 5. 后端模块规划

- Auth：微信静默登录、会话、用户初始化。
- Users：用户资料、关注、粉丝、状态管理。
- Works：作品、草稿、发布、编辑、浏览、点赞、收藏、举报。
- Generate：AI 生成任务、队列、KIE 调用、回调、结果入库。
- Assets：图片上传、OSS 访问、临时 URL、资源归档。
- Credits：积分余额、扣减、奖励、流水。
- Payments：微信支付订单、回调、退款预留。
- Membership：会员方案、权益和有效期。
- Checkin：签到、连续签到和里程碑奖励。
- Invite：邀请关系和奖励。
- Review：微信内容审核、人工审核开关、审核记录。
- Ops：轮播、玩法、风格、分类、热搜、模型、比例、精度配置。
- Messages：系统通知、互动消息、审核消息、公告。
- Feedback：用户反馈、处理状态和回复。
- Admin：后台身份、权限和运营 API。

## 6. KIE.AI 接入

所有目标模型走 KIE Market 任务接口：

- 创建任务：`POST /api/v1/jobs/createTask`。
- 查询任务：`GET /api/v1/jobs/recordInfo?taskId=...`。
- 鉴权：后端请求使用 `Authorization: Bearer <KIE_API_KEY>`。
- 生产优先传 `callBackUrl`，使用回调更新任务状态；轮询只作为兜底。
- 生成完成后应尽快下载或转存结果到 OSS，避免外部结果链接过期。

模型规划：

- GPT Image 2：支持文生图和图生图。
- Nano Banana 2：按 KIE 参数支持生成，保留图片输入参数。
- Nano Banana Pro：按 KIE 参数支持图生图。
- Seedream 4.5：支持文生图；图片编辑能力后续确认后再接入。

生成任务状态统一映射：

- `waiting` / `queuing`：排队中。
- `generating`：生成中。
- `success`：成功。
- `fail`：失败。

## 7. 数据与队列

- PostgreSQL 存储用户、作品、订单、积分流水、生成任务、配置和审核记录。
- Redis 用于会话缓存、限流、任务状态缓存和 BullMQ。
- BullMQ 处理 AI 生成、OSS 转存、审核、通知等异步任务。
- 所有积分扣减和订单状态变更必须由后端事务或幂等逻辑保护。

## 8. 管理后台技术边界

- 管理后台为独立 Web 项目，不嵌入小程序工程。
- 后台通过同一 NestJS API 交互。
- 列表、详情、编辑、启停、审核、通知等能力按原型模块逐步迁移。
- mock 开关关闭后，后台只使用真实 API。

## 9. 部署规划

docker-compose 服务规划：

- `api`：NestJS 后端。
- `admin`：管理后台静态站点或 Web 服务。
- `postgres`：数据库。
- `redis`：缓存与队列。
- `nginx`：域名入口、HTTPS、静态资源和 API 反向代理。

公网域名：

- API 回调路径：`https://ejoyflie.cloud/api/generate/callback`。
- 后续部署时小程序 API、管理后台和静态资源路径由 nginx 统一转发。

# 微信小程序一级页切换白闪：一次从“改主题”到“单页壳”的排查

在一个 uni-app 微信小程序中，我们遇到了一个很典型的问题：

- 首页切到广场、画廊、我的时，偶尔会闪一两帧白屏；
- 深色模式下更明显，白闪会破坏整个切换动画；
- 真机调试和预览的表现还不完全一致。

最后的结论很简单：**这不是页面背景色没配对，而是一级页跨 WebView 切换时出现了原生绘制空档。**

## 先说结论

如果一级入口之间要求稳定、丝滑、没有白闪，不要让它们频繁走 `uni.switchTab`。更可靠的做法是：

1. 保留一个真实的首页页面作为壳；
2. 把广场、画廊、我的做成壳内组件；
3. 一级导航只修改响应式状态，不切换微信页面；
4. 二级页面仍然正常使用 `navigateTo`。

这样一级页切换发生在同一个页面、同一个 WebView 内，白闪的根源就被绕开了。

## 为什么改背景色没有彻底解决？

一开始，我们做了这些常规处理：

- 在 `pages.json` 配置页面背景色；
- 使用微信深色模式配置，让背景随系统主题变化；
- 设置导航栏、底部栏和全局页面背景；
- 在页面生命周期中同步主题颜色。

这些配置没有错，而且应该保留。它们能让**已经创建出来的页面**颜色正确。

但白闪发生得更早：旧页退出、新页 WebView 初始化、首批视图提交之间，微信原生容器会短暂露出默认底色。此时 Vue、CSS 和页面生命周期都还来不及绘制，所以再精确的主题变量也无法完全覆盖这一帧。

## 为什么真机调试和预览不一样？

这是排查时最容易误判的地方。

- 真机调试可能复用部分运行环境，因此第一次切换看起来正常；
- 预览包的页面初始化路径更接近用户实际打开小程序的情况，白闪更容易稳定复现；
- 同一台手机上，首次切换和后续切换也可能不同。

因此，不能因为真机调试“偶尔没闪”就判定问题已经解决。应以预览包、多次互切、深色模式下的结果为准。

## 如何确认是不是 WebView 切换问题？

可以记录路由完成和首屏渲染耗时。如果页面首屏只需要几十到一百多毫秒，但仍能看到白闪，通常说明瓶颈不在业务接口或瀑布流渲染，而在页面容器切换本身。

还可以做一个很直观的验证：

> 把两个一级页临时放进同一个父页面，用 `v-show` 切换。如果白闪消失，基本就可以确认是跨 WebView 的问题。

## 最终方案：一级页单页壳

本项目最终保留 `pages/home/index` 作为唯一的一级页物理壳，广场、画廊、我的作为组件挂进去。

```ts
// primaryShell.ts
export type EmbeddedPrimaryTab = "home" | "plaza" | "gallery" | "mine";

export const activeEmbeddedPrimaryTab = ref<EmbeddedPrimaryTab>("home");
```

首页壳中预挂载一级页组件，再用 `v-show` 控制显示：

```vue
<HomePage v-show="activeEmbeddedPrimaryTab === 'home'" />
<PlazaPage v-if="plazaMounted" v-show="activeEmbeddedPrimaryTab === 'plaza'" />
<GalleryPage v-if="galleryMounted" v-show="activeEmbeddedPrimaryTab === 'gallery'" />
<MinePage v-if="mineMounted" v-show="activeEmbeddedPrimaryTab === 'mine'" />
```

一级导航不再直接切 Tab，而是在当前物理页面是首页壳时改状态：

```ts
export function goRootTab(url: string) {
  if (currentRoute() === "/pages/home/index") {
    if (url === "/pages/plaza/index") {
      activeEmbeddedPrimaryTab.value = "plaza";
      return;
    }
    // 画廊、我的同理
  }

  uni.switchTab({ url });
}
```

为了兼容原先依赖 `onShow` 的页面，组件化后需要在 `onMounted` 和当前 Tab 变为激活状态时主动刷新数据：

```ts
onMounted(refreshPage);

watch(activeEmbeddedPrimaryTab, (tab) => {
  if (tab === "mine") refreshPage();
});
```

刷新函数仍然保留原有的缓存时间判断，避免每次点 Tab 都重复请求接口。

## `v-if` 和 `v-show` 怎么选？

- `v-if`：控制是否首次创建组件。适合首次空闲时再挂载，避免首页首屏负担过重。
- `v-show`：控制已创建组件的显示与隐藏。适合一级页来回切换，能保留滚动位置、列表数据和组件状态。

实际做法是两者结合：首次用 `v-if` 延迟挂载，挂载后用 `v-show` 切换。

## 这套方案的边界

这不是“所有页面都塞进一个页面”。

- 适合：首页、广场、画廊、我的这类高频一级入口；
- 保持独立：创作、详情、设置、消息等二级页面，仍用正常路由跳转；
- 注意：一级页数量太多、每页都很重时，需要延迟预挂载或按需释放，不能无节制常驻。

## 最后的经验

遇到小程序切页白闪时，先分清两类问题：

1. **颜色不一致**：用 `pages.json`、深色模式和原生导航栏配置解决；
2. **原生页面切换空档**：用同一 WebView 内的单页壳解决。

前者是“把背景配对”，后者才是“让白色底板没有机会出现”。

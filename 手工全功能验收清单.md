# 露米绘画手工全功能验收清单

> 用于用户手工复验。自动化验收入口仍是 `corepack pnpm verify:acceptance`；手工验收重点看原型还原、真实交互体验、异常提示和业务闭环。

## 1. 启动与准备

### 1.1 推荐先跑自动化基线

```powershell
corepack pnpm verify:acceptance
```

通过后再开始手工验收。若只想快速确认构建和契约：

```powershell
corepack pnpm verify:static
```

### 1.2 启动小程序 H5

连接真实服务器 API：

```powershell
$env:LUMI_DEV_API_TARGET="http://122.51.235.145:3000"
corepack pnpm dev:mobile:h5
```

通常打开：

```text
http://localhost:5173/
```

如果端口被占用，以终端输出为准。

### 1.3 启动管理后台

另开一个 PowerShell：

```powershell
$env:LUMI_DEV_API_TARGET="http://122.51.235.145:3000"
corepack pnpm dev:admin
```

打开：

```text
http://localhost:5174/
```

测试管理员账号仍是当前服务器测试账号：

```text
admin / admin123
```

### 1.4 mock 开关验收原则

- 小程序：先在 mock 开启状态看界面完整度，再关闭 mock 验证真实接口持久化。
- 管理后台：设置页关闭模拟数据后，所有核心操作应走真实服务器 API。
- 真实支付/KIE 生产验收前，服务器仍允许 mock 支付和 mock 生成；正式上线前必须切换生产配置。

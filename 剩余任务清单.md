# 露米绘画剩余任务清单

> 当前代码功能和自动化验收已基本收口。以下剩余项主要是生产上线配置、真实支付/真实生成的小流量验收，以及最终人工验收。

## 1. 生产安全配置

- [ ] 将服务器 `apps/api/.env` 中 `AUTH_ALLOW_MOCK_LOGIN` 改为 `false`。
- [ ] 将服务器 `apps/api/.env` 中 `GENERATE_ALLOW_MOCK` 改为 `false`。
- [ ] 将服务器 `apps/api/.env` 中 `PAYMENT_ALLOW_MOCK` 改为 `false`。
- [ ] 设置非默认 `ADMIN_PASSWORD`，并确认新密码可以登录管理后台。
- [ ] 收紧 `CORS_ORIGINS`，移除 localhost、127.0.0.1、`[::1]`，只保留生产域名。
- [ ] 确认 `JWT_SECRET`、`ADMIN_JWT_SECRET`、`CALLBACK_SECRET` 均为 32 位以上非默认强密钥。

## 2. 微信支付 V3 配置

- [ ] 准备 `WX_PAY_API_V3_KEY`。
- [ ] 准备 `WX_PAY_CERT_SERIAL_NO`。
- [ ] 准备商户私钥，填写 `WX_PAY_PRIVATE_KEY` 或配置 `WX_PAY_PRIVATE_KEY_PATH`。
- [ ] 准备微信支付平台证书，填写 `WX_PAY_PLATFORM_CERTIFICATE` 或配置 `WX_PAY_PLATFORM_CERTIFICATE_PATH`。
- [ ] 配置 `WX_PAY_NOTIFY_URL=https://ejoyflie.cloud/api/payments/wechat/notify`。
- [ ] 重启 API 后创建一笔小额充值订单，确认能拉起微信支付。
- [ ] 支付成功后确认微信回调验签通过、订单状态变为 paid、积分到账且流水正确。
- [ ] 退款功能如要上线，需要补充微信支付退款接口、退款证书/权限配置和后台退款审核流程；当前后台只提示生产退款配置要求，不会假装退款成功。

## 3. KIE 真实生成验收

- [ ] 关闭 `GENERATE_ALLOW_MOCK` 后，确认 KIE API Key 可用。
- [ ] 用 GPT Image 2 文生图创建一笔小额真实生成任务。
- [ ] 验证 KIE callback 能到达 `/api/generate/callback`。
- [ ] 验证生成结果转存 OSS。
- [ ] 验证生成成功扣积分，失败退积分。
- [ ] 验证生成完成后小程序创作页和画廊任务卡片提示“已自动保存为草稿”。

## 4. 微信登录与小程序真机

- [ ] 在微信开发者工具打开 `apps/mobile`。
- [ ] 使用真实微信 AppID。
- [ ] 关闭 mock 登录后，使用 `wx.login` 完成静默登录。
- [ ] 验证 token 刷新、退出登录、未登录态拦截正常。
- [ ] 验证订阅消息/分享/支付等 H5 无法完全覆盖的微信端能力。

## 5. OSS 与内容审核

- [ ] 确认 OSS bucket 权限、跨域、回源和访问域名策略。
- [ ] 使用真实图片上传，确认预签名 PUT、上传完成、作品发布链路正常。
- [ ] 确认微信文本/图片内容安全接口配置。
- [ ] 打开人工审核开关后，验证作品进入待审核，后台通过后小程序可见。

## 6. 人工最终验收

- [ ] 按 `手工全功能验收清单.md` 跑完小程序手工验收。
- [ ] 按 `手工全功能验收清单.md` 跑完管理后台手工验收。
- [ ] 跑自动化全量验收：

```powershell
corepack pnpm verify:acceptance
```

- [ ] 跑生产安全闸门：

```bash
ENV_FILE=apps/api/.env corepack pnpm check:production-safety
```

- [ ] 服务器 API 健康检查：

```bash
curl -fsS http://127.0.0.1:3000/api/health
```

- [ ] 公网域名健康检查：

```text
https://ejoyflie.cloud/api/health
https://www.ejoyflie.cloud/api/health
```

## 7. 当前已知边界

- 当前功能验收环境仍使用测试管理员 `admin/admin123`，正式上线前必须更换。
- 当前服务器生产安全闸门未通过，原因集中在 mock 开关、CORS、管理员密码和微信支付 V3 配置。
- Windows 本机访问 `https://ejoyflie.cloud` 可能出现 TLS/网络问题，自动化真实接口测试优先使用 `http://122.51.235.145:3000/api`。
- H5 可以验收大部分业务闭环，但微信支付、`wx.login`、微信真机分享等能力必须在微信开发者工具或真机里最终确认。
